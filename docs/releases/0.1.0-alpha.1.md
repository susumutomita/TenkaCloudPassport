Source Candidate の strict SemVer は `0.1.0-alpha.1` です。iOS の `CFBundleShortVersionString` は
pre-release suffix を許さないため、`app.json` の `expo.version`（user-facing Native Version）は

バックアップ、Local Diagnostics、Privacy-preserving Measurement、Facilitator Kit を含む Source-only Candidate です。

## Apple と Expo の境界

2026-07-18 に公式資料を確認しました。

- Apple Account と Xcode Personal Team では、自分の Personal Device へ local build を install して test できる。
  Personal Team の App ID、Device、Provisioning は短期制約があり、継続配布経路ではない。
- TestFlight、App Store Connect、世界向け配布は Apple Developer Program の membership 境界である。
- Expo Go は同梱済み Native module だけを使う。Custom Native Code は Development Build が必要である。
- EAS が署名する iPhone device build は有料 Apple Developer Account が必要である。無料範囲は macOS + Xcode の
  local personal-device build であり、第三者への Public iOS 配布ではない。

## Rollback

- Draft Candidate が不正なら公開せず、既存 Draft と出力を隔離し、新しい未作成 Path で再生成する。
- 公開済み Version の File や checksum は上書きしない。Release を非推奨として理由と利用者 Action を記載し、
  修正版は Version を上げて新しい Candidate / Review / Evidence を作る。
- App Runtime に Migration は追加しないため、Source Release tooling の Rollback は Merge Commit の revert と
  Draft Workflow の停止で行う。
