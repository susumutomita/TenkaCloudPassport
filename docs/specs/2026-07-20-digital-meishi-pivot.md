# 自己紹介カードピボット 仕様書

## 概要

プロダクトの軸を「名刺の否定」から「**無料で渡せる自己紹介**」へ転換する。原点の課題は
「イベントで LT した後に名刺交換したいのに名刺を作っていなかった。プレーリーカード等の
有料サービスにお金を払いたくない」である。名刺という形式が目的ではなく、核は次の 3 点。

1. 名刺交換のタイミングで**連絡先の交換と自己紹介**ができる。
2. **会話のきっかけ**になる（何をやっている人かが伝わる）。
3. **相手と簡単に**交換できる（相手にアプリを要求しない）。

本質は「何をやっている人か知ってもらい、次の打ち合わせ的な接点を持てること」。

## ロードマップ（プロダクトビジョン）

実装は Step 1 から積む。Step 2〜4 は構想であり、LP・README では「構想」と明示して
能力を先取り表示しない（fail-closed）。

| Step | 内容 | 状態 |
| --- | --- | --- |
| 1 | 自己紹介カードを QR で渡す。相手はアプリ不要・標準カメラで連絡先登録。 | 本仕様の実装範囲 |
| 2 | 受け取った側（アプリあり）が相手をインターネットで自動リサーチし、オントロジー（人物・活動・関心の構造）を作る。 | 構想。「外部と通信しない」現行方針の転換を伴うため、着手時に ADR で opt-in 設計を定める |
| 3 | イベント参加者の中で「やっていることが近い人」を可視化する。 | 構想 |
| 4 | 相手もアプリを入れたら、端末内 LLM エージェント同士が自己紹介を交換して接点（次の打ち合わせにつながる話題）を見つける。 | 構想。既存の Pet / Lounge / Bridge 資産がここで主役に戻る |

## ユーザーストーリー

- LT 登壇者として、発表直後に話しかけてくれた人へ、スマホを見せるだけで連絡先と
  「何をやっている人か」を渡したい。
- 受け取る側として、アプリを入れずに標準カメラで QR を読むだけで連絡先登録したい。
- 利用者として、お金もアカウント登録も払いたくない。データは端末の外に出したくない。

## 決定事項（ヒアリング承認済み）

- QR の中身は **vCard 3.0 の直埋め**。相手は iPhone / Android の標準カメラで読むだけで
  連絡先登録できる。サーバー不要・オフライン・完全無料で成立する。
- 名刺の項目: 名前・肩書き・所属 / GitHub・X・SNS・Web リンク / メールアドレス /
  電話番号（各項目は任意。空なら vCard に含めない）。加えて**自己紹介・いまやっている
  こと**（自由入力、会話のきっかけの核。vCard の NOTE に載せる）。
- 既存の Pet / Lounge / Bridge は**導線から外す**（コードは削除しない。ロードマップ
  Step 4 で端末内 LLM エージェント同士の接点探しとして再登場する予定の資産）。
  アプリのメインフローは「自己紹介カード作成 → カード表示（QR）」になる。

## 受け入れ基準

- [ ] 起動 → 名刺の作成・編集（全項目任意、端末内のみ保存）→ 名刺表示画面（大きな
      実 QR + 名前・肩書き・自己紹介）が最短導線になる。
- [ ] QR は ADR-0024 の純 TS エンコーダで生成した**スキャン可能な実 QR**（vCard 3.0）。
      iPhone 標準カメラで読み取ると連絡先追加が提案されることを実機で確認する。
- [ ] vCard は 1,024 byte 以内（encoder 上限）。超過時は保存前に型付きエラーで知らせ、
      どの項目を削れば収まるかが分かる。
- [ ] アカウント・サーバー・テレメトリなしを維持。名刺データは端末内のみ、持ち出しは
      既存の JSON Backup 経路。
- [ ] privacy 方針の転換（氏名・連絡先の**保持を許可**。ただし共有は Owner が QR を
      見せる操作だけ）を ADR で supersede し、privacy データ台帳を更新する。
- [ ] LP と README を新しい軸（無料デジタル名刺・アプリ不要で受け取れる・端末内完結）
      へ書き換える。
- [ ] カバレッジ 100%・`make before-commit` 全緑。

## スコープ外（今回やらない）

- Pet / Lounge / Bridge のコード削除・再設計（導線から外すのみ）。
- 相互交換（相手の名刺を自分が受け取る機能）。まず「渡せる」を完成させる。
- プロフィール Web ページ（URL 方式）・NFC・デザインテンプレート複数対応。
- Local LLM 連携。
